## 实施

### 关键 bug 修复（4 个）

1. **pointSize 公式过小** — `(300 / -mv.z)` 在距离 6500 时只给 0.046px，完全不可见。**修**：常数 300 → 25000，在 distance 3500 处给 7+ px。
2. **Camera 朝向反** — FlyControls 默认 dir = `(sin(yaw)cos(pitch), sin(pitch), cos(yaw)cos(pitch))` 在 yaw=pitch=0 给 (0,0,+1)，与 Three.js 默认 camera 看 -Z 矛盾。**修**：z 分量加负号，dir = `(..., ..., -cos(yaw)*cos(pitch))`。
3. **Plane 不匹配** — CoordinateRings / Landmarks 之前铺在 XZ 平面（y=0），但 PersonStars 数据在 XY 平面（z≈0）。**修**：ring/mansion/landmark 改 z=0 平面，cylinder 旋转 π/2 让 cap 朝 +Z（俯视看是干净圆形）。
4. **Points 拾取不能用 `e.instanceId`** — `e.instanceId` 只对 InstancedMesh 有意义；Points（非 InstancedMesh）应该用 `e.index`。**修**：把 onClick/onPointerMove 从 `pickRef`（hidden，raycaster 跳过）搬到 `displayMat` 同一组 points，删掉整个 pick shader / pick geometry / encodePickId。

### 坐标系统一

数据 layout（precompute_layout.py）：
- `x, y` = disc 2D 平面（r*cos θ, r*sin θ），范围 [-1500, 1500]
- `z` = 世代高度（gen * 8 + jitter ±1.5），范围 [-1.5, 17.5]

Camera + 所有 group 旋转：绕 Z（XY 平面 spin），不是 Y。
- Galaxy.tsx: `group.rotation.z = galaxySpin.angle`（原 rotation.y）
- PersonStars.tsx: `group.rotation.z = galaxySpin.angle`（原 rotation.y）
- CoordinateRings.tsx: `group.rotation.z = galaxySpin.angle`（已对）
- Landmarks.tsx: `group.rotation.z = galaxySpin.angle`（已对）

### 朝代过滤

`store.filters.dynasty` 设置后，PersonStars useEffect 写 `aSize` attribute：
- 命中朝代：size = gen 0→2.0 / gen 1→1.5 / gen 2→1.0
- 不命中：size = 0（vertex shader 仍计算但 gl_PointSize = 0，gl 不画）

不重建 geometry，165k attribute 改写 ~6ms 完成。

### 拾取 threshold

Canvas 加 `raycaster={{ params: { Points: { threshold: 15 } } }}` — Three.js Points raycaster 默认 threshold=1（1 NDC 单位），165k 7+ px 的点完全碰不到；15 px 阈值让 hover/click 容错。
