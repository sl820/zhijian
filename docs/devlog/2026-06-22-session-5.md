# 志鉴 v2 Session 5 (2026-06-22) — Phase D-9 SearchPanel

## 目标
基于 Phase D 完成的 3D 星系（165k 族人节点 + 4 重圆 + 28 宿 + 16 朝代壳 + 朝代过滤 + 永久链接），
加一个 4 tab 的 SearchPanel，让用户能主动搜先祖（姓/名/支系/朝代），点击结果 fly 到该节点并弹出 PersonPanel。

## 关键决策

### 数据层
- `searchSurname(query, limit)` 走 `family_name` 前缀匹配。**数据坑**：`family_name` 字段是 **pinyin**（如 'han'），不是中文（'韩'）。
  解决：在 `ensurePersonsLoaded` 第一次加载时一次性建 `cnToPinyin: Map<string, string>`，从每个 Person 的 `name[0]` → `family_name.toLowerCase()`。
  165k 节点后约 902 个 unique CN 姓字符。
- `searchByName(query, limit)` — `name` / `name_alt` / `courtesy` / `pseudonym` 子串匹配。
- `searchByLine(query, limit)` — `family_uri` / `family_name` 子串匹配；按 `family_uri` 去重只留始祖（generation 最小）。
- `getAllFamilyNames(limit)` — 给姓 tab 自动补全的频次倒序列表。
- `getDynastyCounts()` — 给朝代 tab 的人数统计。
- `personWorldPos(pid)` — 用 layout `Position.x/y/z`（已预计算好），不现场算 `r * cos θ`。

### 性能
- `ensurePersonsLoaded` 之前是 `for await (b of idx.buckets)` 串行拉 83 桶（~50MB），实测 > 6s。
  改成 `Promise.all(buckets.map(fetchBucket))` 并行后 ~1.5s 拉完 + 解析。
  165k 节点进 `Map<string, Person>` 后内存约 80MB（V8 dev tools 实测）。
- 单次 `searchSurname` 扫 165k ≈ 80-120ms（Map.values() + startsWith）。

### 状态层
- 新增 `searchOpen: boolean` + `setSearchOpen(v)`。
- 已有 `flyTarget: {x,y,z} | null` 字段，原来没人写。FlyControls 现在 subscribe 它并 lerp 相机过去。

### UI 层
- `SearchPanel.tsx`（新建）：
  - 顶部 header「搜先祖」+ 关闭按钮
  - 4 tab（姓/名/支系/朝代）
  - 姓/名/支系 tab 一个 input；朝代 tab 直接列 16 朝代 + 人数
  - 结果列表：Person 行显示 name + family_name + 朝代 + 世
  - top:80 left:20 width:360，maxHeight calc(100vh-100px)，result list 60vh 可滚动
- `HUD.tsx` 加「搜」按钮（与重置/隐并排）。
- `App.tsx` mount SearchPanel + 加 `Esc` 关闭 + `/` 打开全局快捷键。

### Camera fly
- FlyControls subscribe `store.flyTarget`：
  - 写入时缓存到 `useRef`
  - useFrame 里 lerp 相机位置到目标
  - 距离 < 5 单位视为到位，清空
  - 缓动速率 FLY_LERP=4.0（约 0.5s 到位）
- 用户拖拽 / 滚轮 / 再按 WASD 自动取消 fly（cancelFly 公共函数）。
- 选人时 z+800 让目标在视野正中（避免太近看不到）。

## 踩坑

1. **family_name 是 pinyin 不是中文**：第一次实现以为 `name[0]` 就是 `family_name`，结果搜「韩」全空。
   调试发现 `family_name = 'han'`。加 cnMap 后解决。
2. **TypeScript `RaycasterParameters` 类型严格**：`<Canvas raycaster={{ params: { Points: { threshold: 15 } } }}>` 报缺 Mesh/Line/LOD/Sprite。`as never` 解决。
3. **layout Position 字段名**：`position.ts` 里之前用 `p.r * Math.cos(p.angle)` 算 x/y，但 layout 里已经有 `p.x` / `p.y` / `p.z` 预计算好（v1 的 npz_to_layout_json.py 写入时算过一次）。直接读更省 + 避免浮点误差。
4. **dev server 不自动重启**：HMR 失败时旧代码还在跑。`taskkill` 杀进程后让 Playwright 重启 vite。
