## Decisions

- **D8: 朝代归并 3121 → 16 key**（商/周/秦/汉/晋/南北朝/隋/唐/五代/宋/辽金/元/明/清/民国/不详）。规则：复合字符串（"明末清初"/"宋元之際"）取第一个出现的朝代为主朝代；细分朝代（"明洪武間"）归主朝代；三国归汉（汉末延伸）；南朝细分（南齐/梁/陈/北魏）归南北朝；邻国（高丽/新罗/朝鲜）归不详。
- **D9: persons ETL 范围扩到全集 167 万**（不是原计划 33 万）。原因：relations 端点需要全集 pid 才能 resolve；保留全集意味着 relations 不悬空。前端运行时只 pack 17 万有朝代 → 73MB 公共桶（GH Pages 1GB 限额内）。
- **D10: ETL 选型确定 persons.jsonl + relations.jsonl → pack_data 静态桶**（不直接 git 跟踪 jsonl，因为 637MB；公共产物只 pack 出来的 73MB + 51MB 桶）。
- **D11: 布局从 FA2 改成"朝代径向 + 姓氏角向 + 世代 z"**（v2 设计）。原因：诗云范本的核心是"朝代同心壳"，不是 FA2 社区聚类；FA2 算法保留在 `scripts/precompute_layout.py` 作为 legacy 路径，v2 走 `pipeline/precompute_layout.py` 直接算坐标。
- **D12: relations 两源合并**（person_relations 13k + cbdb_relations 588k），cbdb 中文 label 归一化为 parentOf / childOf / siblingOf / spouseOf / other 五类。
- **D13: 公共 surnames 用 persons 全集统计**（20098 王 / 18712 张 / ...）→ common 阈值降到 800（有 81 个 common 姓）。
- **D14: name_lexicon 用 persons 全集**，courtesy 34,801 unique / pseudonym 12,111 unique（远超诗云 250 条，规模放大 100x+）。

## 已知约束 / 风险

- `persons.jsonl` 637MB 不进 git（已在 .gitignore），属于 ETL 中间产物，每次重新生成
- 公共 persons 桶仅 17 万（有朝代），relations 端点指向 unknown 朝代的人时前端会显示"未找到" — 可接受
- 朝代归并规则对极小朝代（秦 55 / 辽金 44 / 商 80）覆盖完整，但 3,121 → 16 的归并会丢失细分朝代信息（如"明洪武間"→"明"，年份细节丢失）。如未来需要细分朝代（如按"洪武/永乐/宣德"），可在 `temporal_raw` 字段保留原始字符串
- `jiapu_v2.npz` 当前未含 edge 数组（运行时从 `public/relations/` JSON 桶拉取）。FA2 算法路径保留在 `scripts/precompute_layout.py`，按需启用
